üß™ Testing Steps

Step 1: Start the Server

cd /Users/macbookair/Desktop/PROJECT/projectx
python manage.py runserver

Step 2: Test Regular User Registration

  1. Go to: http://localhost:8000/accounts/register/
  2. Select "Regular User"
  3. Fill in the form and register
  4. Check that "Add Product" link is NOT visible in navigation

Step 3: Test Admin Registration

  1. Go to: http://localhost:8000/accounts/register/
  2. Select "Admin"
  3. Enter admin password: 11814
  4. Fill in the form and register
  5. Check that "Add Product" link IS visible in navigation

Step 4: Test Product Creation

  1. Login as admin user
  2. Click "Add Product" in navigation
  3. Fill in product details and upload image
  4. Submit and verify product is created

Step 5: Test Access Control

  1. Login as regular user
  2. Try to access: http://localhost:8000/products/add-product/
  3. Should be redirected with "Access denied" message

  • Only admin users can access product creation form
  • Regular users get "Access denied" message when trying to add products
  • Admin password verification works (only 11814 is accepted)

‚ùå Troubleshooting

If migrations fail:

# Try with Python 3.11 or 3.12
python3.11 manage.py makemigrations accounts
python3.11 manage.py migrate

If admin registration fails:

  • Check that admin password is exactly "11814"
  • Verify all form fields are filled
  • Check Django server logs for errors

If "Add Product" doesn't appear:

  • Make sure you're logged in as admin
  • Check that user.is_admin is True in database
  • Clear browser cache and refresh
